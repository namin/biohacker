BioHacker: The Network Coverage Problem

Given

Beliefs holding over all organisms

We believe in a set of chemical reactions, converting reactants (inputs) into products (outputs). These reactions are organized into pathways, typically catabolisms, converting nutrients into intermediary compounds, and anabolisms, converting intermediary compounds into essential compounds. The notion of converter from a set of inputs to a set of outputs generalizes over reactions and pathways. A converter is either primitive (a reaction) or compound (a pathway). A compound converter is a graph, where each node represents a converter; each directed edge, an inner input-output dependency; each dangling incoming edge, an input; and each dangling outgoing edge, an output. With these primitives (reactions), means of combinations (pathways) and means of abstractions (converters), we organize our set of reactions hierarchically.

Assumptions per organism

We assume an organism is capable of catalyzing a subset of our universal set of reactions, because an organism has genes, and genes codes for proteins, and proteins forms enzymes, and enzymes control reactions.

Beliefs holding per organism

For each organism, we believe in a set of compounds essential for growth of the organism. Crucially, we collect and also believe some growth / no-growth experiments. In addition to an outcome (growth / no-growth), an experiment specifies the presence/absence of nutrients and genes.

Show

Underestimations / Completeness

If we enable the assumptions of a growth experiment, and we cannot produce all essential compounds, we have a contradiction: our assumptions of the organisms must contain false negatives, i.e. some additional universal converters should be assumed.

Overestimations / Coherence

If we enable the assumptions of a no-growth experiment, and we can produce all essential compounds, we have a contradiction: our asssumptions of the organisms must contain false positives, i.e. some assumptions must be retracted.

Efficienctly

By organizing the converters hierarchically, we can consider the higher-level converters first, delving into their internals only when necessary or requested.
